#43 - Add SR 661.11 to the legal corpus
What & Why
The entitlement catalog in backend/seed/entitlements.json contains citations to SR 661.11 (Verordnung über die Wehrpflichtersatzabgabe — exemption-tax ordinance), but neither the manual seed nor the Fedlex snapshot covers it. Right now those entitlements either fail retrieval or fall through to a generic suppression message. This is a silent correctness bug: the user never learns the right exists.

Done looks like
Every article cited by an SR-661.11 entitlement is present in the ingested corpus (DE original-language text), with effective_date and repealed_date populated correctly.
A spot-check /scan against a profile that triggers an SR-661.11 entitlement returns a verified benefit (not "requires_evidence_review") with the correct citation surfaced.
Adding 661.11 to the ingester's default SR list (or run command) is documented so the next refresh picks it up automatically.
Out of scope
Adding new entitlements that cite SR 661.11 — only filling the gap for the ones that already do.
Translating to FR / IT / EN — DE authoritative is sufficient for the retrieval guardrails.
Steps
Audit entitlements.json to enumerate exactly which 661.11 articles are cited, so the ingester targets the right scope.
Run the Fedlex ingester for SR 661.11 and merge its records into the snapshot file produced by the previous task.
Verify the retrieval guardrails accept the new records (canton = "CH", effective_date <= today, repealed_date either null or future).
Update the smoke test or add a fixture profile that exercises one 661.11 entitlement so the regression is caught by the offline test suite.
